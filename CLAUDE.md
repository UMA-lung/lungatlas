# LungAtlas

肺がん臨床医向けオープンテキストブック。純粋な静的HTMLサイト（ビルドツールなし）。

## 絶対に守ること
- 既存ファイルを削除しない（git rm も禁止）
- ファイルを上書きする前に必ず内容を確認し、確認を求めること
- pushの前に必ず「pushしてよいですか？」と確認すること

## ファイル構成

```
index.html              # 変異陰性NSCLC化学療法
driver-positive.html    # 変異陽性NSCLC（日本語）
driver-positive-en.html # 変異陽性NSCLC（英語）
egfr-firstline.html     # EGFR変異陽性NSCLC 1次治療選択ガイド
anatomy.html            # 肺の解剖学・CT画像診断
about.html              # このサイトについて（日本語）
about-en.html           # About（英語）
nav.html                # 全ページ共通ナビ（変更時は全ページへの影響を確認）
home.html               # トップページ（未公開・注意）
Textbook/               # 参考PDFテキスト集（git管理外）
```

## デザインルール
- 新ページはindex.htmlのHTMLスタイルに合わせる
- nav.htmlのナビバーを必ず使用する
- 文字コードはUTF-8
- カラーパレット：グリーン基調（`--green-dark: #0d5c3a`など）
- フォント：Inter（本文）、JetBrains Mono（コード・強調）、Google Fonts経由

## Git運用
- コミットメッセージは日本語でよい
- mainブランチに直接pushしてよい
- pushは必ず医師の確認後に行う（**例外：scraps/ 以下のファイルのみの変更は確認不要で commit・push してよい**）

## scraps → 詳細ページ昇格時のルール
- scraps記事を詳細ページに昇格する際は、vault（vault/trials/ または vault/concepts/）を新規作成し、データを再検証する。
- scraps時点のHTMLは出典確認の下書きとして参照してよいが、数値をそのまま転記コピーせず、原著（PMID/DOI）ベースで再検証すること。
- 昇格でデータが変わった場合、まとめページ（一覧・比較表）の該当数値も確認・修正する。

## 医学的内容
- **書かれていないことを推測で補填しない**：テキスト・論文・ユーザーの入力に明記されていない内容（数値・臨床的意見・手術難易度など）は記載しない。根拠のある情報のみ書く
- 薬剤名・試験名は正確に記載する
- 参照したPDFのページ番号をコメントに残す
- 未確認の情報は「要確認」とコメントする

## 控えて欲しい表現

以下の表現はサイト上で使わない：

| 避ける表現 | 理由・代替 |
|---|---|
| 最推奨 | 推奨グレードは文脈依存。代替：「推奨される」「選択肢となる」 |
| 唯一 | 代替治療の存在を否定しうる |
| 絶対 | 医療判断に絶対はない |
| 第一選択 | ガイドラインの文言を正確に引用する場合のみ使用可 |
| 最強 | 科学的根拠のない誇張 |
| 他レジメンを上回る | 直接比較試験で示された場合のみ記載可。その旨を明記する |
| 固形腫瘍史上最長 | 代替：「標準的選択肢」「有力な選択肢」 |

推奨する代替表現：
- 多くの症例で優先される
- 代表的レジメン
- 有力な選択肢
- 標準的選択肢
- 探索的解析で示唆された（サブグループ・探索的解析の結果に使う）
- 直接比較ではない点に注意
- 症例選択が重要
- 毒性管理に習熟した環境での使用が望ましい

## スキル
新ページ作成・push・PDF読み込みの手順は ~/.claude/skills/ 以下のSkillを参照。

- new-page: 新ページ雛形の作成
- trial-table: 臨床試験比較表の作成
- safe-push: commit・pushの安全手順
- pdf-cite: PDF引用とページ番号コメント管理
- medical-content: 医学執筆ルール（禁止表現・参照書式・必須注記）
- site-check: 公開前QA（構造・リンク・表現・参照の一括点検）
- translate-en: 英語版（-en.html）の作成・同期
- ref-check: 参照文献リンク（PMID/DOI）の抽出・検証
